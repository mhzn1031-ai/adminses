### WebRTC Baƒülantƒ± Akƒ±≈üƒ± (P2P + Server Recording)

```
1. Client ‚Üí Server: POST /api/call/notify
   ‚îî‚îÄ> DB: CallSession (pending)
   ‚îî‚îÄ> Telegram: Notification
   ‚îî‚îÄ> WebSocket: pending_update ‚Üí Admin

2. Admin ‚Üí Server: POST /api/call/respond (accept)
   ‚îî‚îÄ> DB: CallSession (accepted)
   ‚îî‚îÄ> Daily limit check

3. WebRTC Signaling (WebSocket):
   Admin ‚Üí offer ‚Üí Server ‚Üí Client
   Client ‚Üí answer ‚Üí Server ‚Üí Admin
   Both ‚Üí ICE candidates ‚Üí Server ‚Üí Other

4. P2P Connection Established
   ‚îî‚îÄ> Direct audio/video stream

5. Server-Side Recording (Parallel):
   Client ‚Üí POST /api/record/offer ‚Üí aiortc
   Admin ‚Üí POST /api/record/offer ‚Üí aiortc
   ‚îî‚îÄ> MediaRecorder ‚Üí .webm files

6. Call End:
   Either ‚Üí POST /api/call/end
   ‚îî‚îÄ> DB: duration update
   ‚îî‚îÄ> WebSocket: call_ended
   ‚îî‚îÄ> POST /api/record/stop
```
